# 数据监控频率字段对接文档（P0）

本文档面向 **前端数据监控页面**，说明后端 Manager 新增「库表数据频率」字段后的接口约定。  
本次改动用于解决周频 / 月频库表在非产数交易日记录数为 0 时，被误标为「数据异常」的问题。

| 项目 | 路径 |
| --- | --- |
| 前端页面 | `/data-monitor`（数据监控页面） |
| 库表配置列表接口 | `GET /api/ddb-monitors` |
| 创建库表配置接口 | `POST /api/ddb-monitors` |
| 编辑库表配置接口 | `PUT /api/ddb-monitors/:monitorId` |
| 数据概览矩阵接口 | `GET /api/ddb-monitors/snapshots` |
| 后端数据表 | `ddb_monitor.frequency` |

---

## 0. 现状说明（重要）

| 项 | 现状 |
| --- | --- |
| 数据频率 | 之前没有单独字段，所有库表默认按日频判断 |
| 0 记录判断 | 统计到 `row_count = 0` 时会写入 / 展示为异常 |
| 周频数据 | 只有每周最后一个交易日正常产数，其他交易日为 0 是正常情况 |
| 月频数据 | 只有每月最后一个交易日正常产数，其他交易日为 0 是正常情况 |
| DolphinDB 同步 | 同步库表配置时不会自动识别频率 |

因此本次新增 `frequency` 字段：

- 默认全部为日频 `daily`
- 特殊周频 / 月频库表由用户在「库表配置」中手动编辑
- DolphinDB 同步不会覆盖用户手动配置的频率
- 数据概览接口由后端根据频率和交易日自动修正展示状态

---

## 1. 目标行为

数据监控页面新增「数据频率」能力：

```text
打开库表配置
  → GET /api/ddb-monitors
  → 每行展示数据频率

新建 / 编辑库表配置
  → 表单中选择数据频率
  → POST /api/ddb-monitors 或 PUT /api/ddb-monitors/:monitorId 提交

打开数据概览
  → GET /api/ddb-monitors/snapshots
  → 后端返回已修正后的 status
```

前端不需要自己判断「周最后交易日」或「月最后交易日」，继续按接口返回的 `status` 渲染即可。

---

## 2. 频率枚举

| 前端展示 | 推荐提交值 | 后端存储值 | 说明 |
| --- | --- | --- | --- |
| 日频 | `daily` | `daily` | 每个交易日都应产数 |
| 周频 | `weekly` | `weekly` | 每周最后一个交易日应产数 |
| 月频 | `monthly` | `monthly` | 每月最后一个交易日应产数 |

前端下拉建议：

```ts
const frequencyOptions = [
  { label: '日频', value: 'daily' },
  { label: '周频', value: 'weekly' },
  { label: '月频', value: 'monthly' }
]
```

推荐前端统一使用 `frequencyCode` 作为表单字段。

---

## 3. 接口变更：库表配置列表 `GET /api/ddb-monitors`

### 3.1 Query 参数

现有查询参数不变。本次不新增筛选参数。

```http
GET /api/ddb-monitors?page=1&page_size=100&group_id=行情数据&keyword=stock
```

### 3.2 响应字段

库表配置列表每个监控项新增字段：

```json
{
  "monitor_id": "market_data_1w_stock",
  "database": "dfs://market_data_1w",
  "table_name": "stock",
  "date_column": "date",
  "date_format": "YYYY-MM-DD",
  "frequency": "周频",
  "frequencyCode": "weekly",
  "frequency_code": "weekly",
  "frequencyLabel": "周频",
  "frequency_label": "周频"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `frequency` | string | 中文展示值，前端表格可直接展示 |
| `frequencyCode` | string | 英文枚举，推荐用于表单回填和提交 |
| `frequency_code` | string | snake_case 英文枚举 |
| `frequencyLabel` | string | 中文展示值，兼容 label 命名 |
| `frequency_label` | string | snake_case 中文展示值 |

> 推荐：表格展示用 `frequency`；表单回填和保存用 `frequencyCode`。

---

## 4. 接口变更：创建库表配置 `POST /api/ddb-monitors`

### 4.1 请求字段

新增字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `frequencyCode` | string | 否 | 数据频率，推荐传英文枚举 |
| `frequency` | string | 否 | 兼容字段，可传英文枚举或中文 |
| `frequency_code` | string | 否 | snake_case 兼容字段 |

未传时默认：`daily`（日频）。

### 4.2 请求示例

推荐传 `frequencyCode`：

```json
{
  "monitor_id": "market_data_1w_stock",
  "database": "dfs://market_data_1w",
  "table_name": "stock",
  "date_column": "date",
  "date_format": "YYYY-MM-DD",
  "group_id": "行情数据",
  "frequencyCode": "weekly",
  "enabled": 1,
  "description": "股票周频行情数据"
}
```

也兼容传中文：

```json
{
  "database": "dfs://market_data_1m",
  "table_name": "stock",
  "date_column": "date",
  "date_format": "YYYY-MM-DD",
  "group_id": "行情数据",
  "frequency": "月频"
}
```

### 4.3 响应

创建成功响应结构不变，`data` 中会返回频率字段：

```json
{
  "code": 0,
  "message": "Monitor created",
  "data": {
    "monitor_id": "market_data_1w_stock",
    "frequency": "周频",
    "frequencyCode": "weekly"
  }
}
```

---

## 5. 接口变更：编辑库表配置 `PUT /api/ddb-monitors/:monitorId`

### 5.1 请求字段

编辑接口同样支持：

```json
{
  "frequencyCode": "monthly"
}
```

注意：编辑接口当前为整体更新语义，前端提交时应保留原有必填字段，例如：

```json
{
  "database": "dfs://market_data_1m",
  "table_name": "stock",
  "date_column": "date",
  "date_format": "YYYY-MM-DD",
  "group_id": "行情数据",
  "frequencyCode": "monthly",
  "enabled": 1
}
```

### 5.2 表单回显

编辑弹窗打开时，建议使用：

```ts
form.frequencyCode = data.frequencyCode || 'daily'
```

---

## 6. 接口变更：数据概览矩阵 `GET /api/ddb-monitors/snapshots`

### 6.1 请求参数

现有请求参数不变。

```http
GET /api/ddb-monitors/snapshots?group_id=行情数据&date_from=2026-07-01&date_to=2026-07-16
```

### 6.2 响应状态逻辑

后端会根据库表频率自动修正 `status`：

| 频率 | 日期类型 | row_count | 返回 `status` |
| --- | --- | --- | --- |
| 日频 | 任意交易日 | 0 | `zero` |
| 周频 | 每周最后一个交易日 | 0 | `zero` |
| 周频 | 非每周最后一个交易日 | 0 | `success` |
| 月频 | 每月最后一个交易日 | 0 | `zero` |
| 月频 | 非每月最后一个交易日 | 0 | `success` |

也就是说，前端展示时继续使用 `status` 即可，不需要根据 `row_count` 自己二次判断异常。

### 6.3 响应字段

快照项新增调试字段：

```json
{
  "monitor_id": "market_data_1w_stock",
  "database": "dfs://market_data_1w",
  "table_name": "stock",
  "date": "2026-07-15",
  "row_count": 0,
  "status": "success",
  "raw_status": "zero",
  "rawStatus": "zero",
  "frequency": "周频",
  "frequencyCode": "weekly",
  "frequency_code": "weekly",
  "expected_data_day": false,
  "expectedDataDay": false
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `status` | string | 前端展示用状态，已按频率修正 |
| `raw_status` | string | 数据库原始统计状态，仅用于排查 |
| `rawStatus` | string | camelCase 原始状态 |
| `frequency` | string | 中文频率 |
| `frequencyCode` | string | 英文频率枚举 |
| `expected_data_day` | boolean | 当前日期是否为该库表应产数日 |
| `expectedDataDay` | boolean | camelCase 兼容字段 |

> 前端正常展示只使用 `status` 和 `row_count` 即可。`raw_status`、`expected_data_day` 主要用于联调排查。

---

## 7. DolphinDB 同步说明

DolphinDB 库表同步逻辑不会覆盖 `frequency` 字段。

同步新发现的库表时：

- 新库表默认 `frequency = daily`
- 已存在库表保留用户手动编辑的频率

因此前端只需要在库表配置编辑入口提供频率选择，不需要在同步入口处理频率。

## 8. 前端改造清单

| 项 | 说明 |
| --- | --- |
| 库表配置表格 | 增加「数据频率」列，展示 `frequency` |
| 新建库表配置弹窗 | 增加「数据频率」下拉框，默认 `daily` |
| 编辑库表配置弹窗 | 回显 `frequencyCode`，允许修改 |
| 保存请求 | `POST` / `PUT` 时提交 `frequencyCode` |
| 数据概览矩阵 | 继续按接口返回的 `status` 渲染状态 |
| 异常统计 chip | 如果统计来源为 snapshots 接口，应按修正后的 `status` 统计 |

## 10. 数据库迁移

后端已提供迁移脚本：

```text
quant_ops_system/db/migrations/009_ddb_monitor_frequency.sql
```

也可以通过重启 Manager 自动补字段。字段定义：

```sql
frequency VARCHAR(16) NOT NULL DEFAULT 'daily'
```

历史库表默认均为 `daily`，周频 / 月频需要人工在库表配置页面修改。

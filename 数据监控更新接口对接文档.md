# 数据监控更新接口对接文档

本文档给前端说明数据监控「库表配置」页面的编辑、启用、禁用接口调整。

## 背景

后端仅保留两个 POST action 接口处理库表配置更新：

```http
POST /api/ddb-monitors/:monitorId/enabled
POST /api/ddb-monitors/:monitorId/update
```

旧的 `PUT/PATCH /api/ddb-monitors/:monitorId` 已移除，前端不要再调用。

---

## 1. 启用 / 禁用库表

### 接口

```http
POST /api/ddb-monitors/:monitorId/enabled
```

### 请求体

启用：

```json
{
  "enabled": 1
}
```

禁用：

```json
{
  "enabled": 0
}
```

### 响应示例

```json
{
  "code": 0,
  "message": "Monitor updated",
  "data": {
    "monitor_id": "mon_market_data_1w_index",
    "monitorId": "mon_market_data_1w_index",
    "enabled": 0,
    "updated_at": "2026-07-16 14:20:00",
    "updatedAt": "2026-07-16 14:20:00"
  }
}
```

### 前端封装示例

```ts
export function updateDdbMonitorEnabled(monitorId: string, enabled: boolean | number) {
  return request.post(`/api/ddb-monitors/${monitorId}/enabled`, {
    enabled: enabled ? 1 : 0
  })
}
```

成功判断：

```ts
res.code === 0
```

## 2. 编辑库表配置

### 接口

```http
POST /api/ddb-monitors/:monitorId/update
```

### 请求体

请求体沿用原编辑表单字段。

```json
{
  "database": "dfs://market_data_1w",
  "table_name": "index",
  "date_column": "date",
  "date_format": "YYYY-MM-DD",
  "group_id": "market_data",
  "frequencyCode": "weekly",
  "where_extra": "",
  "related_task_ids": [],
  "enabled": 1,
  "description": ""
}
```

### 响应

成功时返回轻量结果。前端不要依赖返回完整监控项，保存成功后刷新当前列表即可。

```json
{
  "code": 0,
  "message": "Monitor updated",
  "data": {
    "monitor_id": "mon_market_data_1w_index",
    "monitorId": "mon_market_data_1w_index",
    "updated_at": "2026-07-16 15:30:00",
    "updatedAt": "2026-07-16 15:30:00"
  }
}
```

### 前端封装示例

```ts
export function updateDdbMonitor(monitorId: string, payload: DdbMonitorUpdatePayload) {
  return request.post(`/api/ddb-monitors/${monitorId}/update`, payload)
}
```

---

## 3. 前端替换清单

| 场景 | 旧接口 | 新接口 |
| --- | --- | --- |
| 编辑库表配置 | `PUT/PATCH /api/ddb-monitors/:monitorId` | `POST /api/ddb-monitors/:monitorId/update` |
| 启用库表 | `PUT/PATCH /api/ddb-monitors/:monitorId` | `POST /api/ddb-monitors/:monitorId/enabled` |
| 禁用库表 | `PUT/PATCH /api/ddb-monitors/:monitorId` | `POST /api/ddb-monitors/:monitorId/enabled` |

旧接口已移除，调用 `PUT/PATCH /api/ddb-monitors/:monitorId` 会返回路由不存在。

---

## 4. 注意事项

- `monitorId` 使用列表返回的 `monitor_id` 或 `monitorId`。
- 成功判断统一使用 `code === 0`。
- 编辑后建议刷新当前页库表列表。
- 启用 / 禁用可以先本地更新当前行状态，再异步刷新列表。

# 数据监控对接文档

本文档面向 **后端 Manager / 数据平台** 团队，说明「数据监控」模块的业务边界、数据模型、REST API 与夜间统计任务建议实现。前端 Demo 已落地，类型与 Mock 实现可作为联调参考。

| 项目 | 路径 |
| --- | --- |
| 前端页面 | `/data-monitor`（`src/views/DataMonitorView.vue`） |
| 类型定义 | `src/types/domain.ts` → `DdbMonitor*` |
| 前端 API | `src/api/ddbMonitors.ts` |
| Mock 实现 | `mock-server/src/routes/ddbMonitors.ts`、`mock-server/src/store/ddbMonitors.ts` |

---

## 1. 业务目标

数据监控用于 **按交易日查看 DolphinDB 库表行数是否正常**，帮助运维快速发现「任务看似成功但表里没有数据 / 行数异常」的问题。

### 1.1 本模块做什么

- 维护 **监控配置**（库、表、日期字段、分组、弱关联任务等）
- 展示 **行数统计快照**（矩阵热力图：行=表，列=日期）
- 支持 **重新统计**（`recount`）：仅重跑 `SELECT count(*) ...`，刷新快照
- 在详情抽屉中 **跳转关联任务**（弱关联，多对多）

### 1.2 本模块不做什么

- **不提供表级数据补跑**（不能「只补当前表某一天的数据」）
- **重新统计不会触发任务执行**；写数仍由任务调度系统负责
- 快照接口 **不返回** `related_task_ids`；前端通过 `monitor_id` 查配置表拼接

### 1.3 与任务的关系

| 概念 | 说明 |
| --- | --- |
| 任务 | 调度系统中的 Python / DAG 执行任务 |
| 监控项 | 一张 DolphinDB 表的 count 监控配置 |
| 关联关系 | **多对多、弱关联**：`related_task_ids` 仅用于 UI 跳转，不参与统计逻辑 |
| 写数 | 任务可能一次写入多张表；无法从表反推唯一任务 |

## 2. 权限

| 权限 code | 说明 | 典型角色 |
| --- | --- | --- |
| `data:view` | 查看监控概览与快照 | your_username、operator、researcher |
| `data:manage` | 新建/编辑/删除监控项、重新统计 | your_username、operator |

前端常量：`PERMISSIONS.DATA_MONITOR_VIEW` / `DATA_MONITOR_MANAGE`（`src/features/auth/permissions.ts`）。

## 3. 数据模型（建议 MySQL）

### 3.1 监控配置表 `ddb_monitor`

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `monitor_id` | varchar(64) PK | 是 | 如 `mon_market_daily_bar` |
| `database` | varchar(128) | 是 | DolphinDB 库，如 `dfs://market` |
| `table_name` | varchar(128) | 是 | 表名 |
| `date_column` | varchar(64) | 是 | 过滤用日期字段，如 `trade_date` |
| `date_format` | varchar(16) | 否 | `YYYY-MM-DD`（默认）或 `YYYYMMDD` |
| `where_extra` | varchar(512) | 否 | 额外 WHERE，不含 `date_column` 条件 |
| `group_name` | varchar(32) | 是 | 业务分组：行情 / 因子 / 财务 / 基础数据 / 其他 |
| `related_task_ids` | json | 否 | 任务 ID 数组，弱关联 |
| `enabled` | tinyint(1) | 是 | `1` 启用，`0` 禁用 |
| `description` | varchar(512) | 否 | 备注 |
| `created_at` | datetime | 否 | |
| `updated_at` | datetime | 否 | |

### 3.2 统计快照表 `ddb_monitor_snapshot`

建议唯一键：`(monitor_id, date)`。

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `monitor_id` | varchar(64) | 是 | 外键 → `ddb_monitor.monitor_id` |
| `database` | varchar(128) | 是 | 冗余，便于列表查询 |
| `table_name` | varchar(128) | 是 | 冗余 |
| `group_name` | varchar(32) | 是 | 冗余 |
| `date` | date | 是 | 交易日 `YYYY-MM-DD` |
| `row_count` | bigint | 否 | **表行数（条数）**；未统计或失败可为 `null`；前端直接展示，无文件大小字段 |
| `status` | varchar(16) | 是 | 见下表 |
| `checked_at` | datetime | 否 | 最近一次统计完成时间 |
| `duration_ms` | int | 否 | 统计耗时 |
| `error_message` | text | 否 | 失败原因 |

#### `status` 枚举

| 值 | 含义 | 前端展示 |
| --- | --- | --- |
| `pending` | 尚未统计 | 灰色「未统计」 |
| `success` | 统计成功且 `row_count > 0` | 绿色 |
| `zero` | 统计成功但行数为 0 | 橙色警告 |
| `failed` | 执行 count 失败 | 红色 |

## 4. 统计 SQL 生成（后端）

对每个 **启用** 的监控项、每个目标交易日 `D`：

```sql
-- 伪代码，按 date_format 拼接日期字面量
SELECT count(*) AS cnt
FROM loadTable('{database}', '{table_name}')
WHERE {date_column} = {formatted_date}
  [AND {where_extra}]
```

注意：

- `where_extra` 由运维配置，后端需做基本校验防注入
- 禁用项（`enabled=0`）夜间任务跳过；手动 recount 可返回 400
- 统计结果写入/更新 `ddb_monitor_snapshot`，并填充冗余字段

---

## 5. 夜间 Cron 任务（建议）

| 项 | 建议 |
| --- | --- |
| 触发时间 | 如每日 02:00（任务跑完后） |
| 输入 | 前一交易日或配置窗口内缺失快照的日期 |
| 并发 | 按 DolphinDB 连接池限制；可分批 |
| 失败 | 写 `status=failed` + `error_message`，便于前端展示 |
| 与任务调度 | **独立进程**，不经过任务执行引擎 |

## 6. REST API

基础路径：`/api/ddb-monitors`  
响应包装：与全局一致 `{ code, message, data }`，`code=0` 为成功。

### 6.1 接口一览

| 方法 | 路径 | 权限 | 说明 |
| --- | --- | --- | --- |
| GET | `/api/ddb-monitors` | `data:view` | 监控配置列表 |
| POST | `/api/ddb-monitors` | `data:manage` | 新建监控项 |
| PUT | `/api/ddb-monitors/:monitorId` | `data:manage` | 更新监控项 |
| DELETE | `/api/ddb-monitors/:monitorId` | `data:manage` | 删除监控项 |
| GET | `/api/ddb-monitors/snapshots` | `data:view` | 快照列表（矩阵数据源） |
| POST | `/api/ddb-monitors/:monitorId/recount?date=` | `data:manage` | 单项重新统计 |
| POST | `/api/ddb-monitors/recount-batch?date=` | `data:manage` | 指定日期全部启用项重新统计 |

> 路由注册顺序：`/snapshots`、`/recount-batch` 须在 `/:monitorId` 之前，避免被路径参数吞掉。

### 6.2 GET `/api/ddb-monitors`

**Response `data`**: `DdbMonitor[]`

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "monitor_id": "mon_market_daily_bar",
      "database": "dfs://market",
      "table_name": "daily_bar",
      "date_column": "trade_date",
      "date_format": "YYYY-MM-DD",
      "group_name": "行情",
      "related_task_ids": ["task_market_stock_daily"],
      "enabled": 1,
      "description": "A 股日频行情主表",
      "created_at": "2026-06-01T10:00:00+08:00",
      "updated_at": "2026-06-20T10:00:00+08:00"
    }
  ]
}
```

### 6.3 POST / PUT 请求体 `DdbMonitorPayload`

```json
{
  "database": "dfs://market",
  "table_name": "daily_bar",
  "date_column": "trade_date",
  "date_format": "YYYY-MM-DD",
  "where_extra": "symbol_type='stock'",
  "group_name": "行情",
  "related_task_ids": ["task_market_stock_daily"],
  "enabled": 1,
  "description": "备注"
}
```

- POST：服务端生成 `monitor_id`（或接受可选 `monitor_id`）
- PUT：路径上的 `monitorId` 为准，不可改主键

### 6.4 GET `/api/ddb-monitors/snapshots`

**Query**

| 参数 | 说明 |
| --- | --- |
| `date_from` | 起始日期 `YYYY-MM-DD` |
| `date_to` | 结束日期 `YYYY-MM-DD` |
| `group_name` | 业务分组过滤 |
| `keyword` | 库名 / 表名模糊搜索 |

**Response `data`**: `DdbMonitorSnapshot[]`

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "monitor_id": "mon_market_daily_bar",
      "database": "dfs://market",
      "table_name": "daily_bar",
      "group_name": "行情",
      "date": "2026-06-27",
      "row_count": 512340,
      "status": "success",
      "checked_at": "2026-06-30T02:15:00+08:00",
      "duration_ms": 156
    }
  ]
}
```

**说明**：快照 **不含** `related_task_ids`；详情抽屉由前端 `GET /api/ddb-monitors` 缓存拼接。

### 6.5 POST `.../recount?date=YYYY-MM-DD`

对单个监控项执行 count，更新快照后返回 **该日单条** `DdbMonitorSnapshot`。

### 6.6 POST `/api/ddb-monitors/recount-batch?date=YYYY-MM-DD`

对该日所有 `enabled=1` 的监控项重新统计，返回 `DdbMonitorSnapshot[]`。

## 7. 前端页面行为（联调参考）

### 7.1 监控概览 Tab

- 矩阵：行=表（含分组、库名），列=日期
- 格子内展示 **`row_count` 完整条数**（千分位格式，如 `512,340`），不做 K/M 缩写；**无文件大小字段**，后端只需统计行数
- 格子颜色：`pending` / `success` / `zero` / `failed`
- 点击格子 → 详情抽屉：记录数、统计耗时、错误、关联任务跳转、**重新统计**
- 工具栏：分组筛选、关键字、刷新、**重算最近一日**（调用 `recount-batch`）

### 7.2 监控配置 Tab

- 表格展示全部监控项
- 有 `data:manage` 权限时可新建 / 编辑 / 删除
- **关联任务**列：展示任务名称（由 `related_task_ids` + `GET /api/tasks` 缓存解析），点击跳转 `/tasks/:taskId`；未配置时显示「未配置」
- 编辑抽屉中关联任务为多选下拉，选项数据来自 `GET /api/tasks`

### 7.3 页面加载

并行请求：

1. `GET /api/ddb-monitors`
2. `GET /api/ddb-monitors/snapshots`
3. `GET /api/tasks`（配置 Tab 关联任务名称解析、详情抽屉跳转）

## 9. 联调检查清单

- [ ] `GET /api/ddb-monitors` 与 `GET /api/ddb-monitors/snapshots` 字段名与 `domain.ts` 一致
- [ ] 快照 `status` 四枚举与前端颜色映射一致
- [ ] `recount` / `recount-batch` 仅执行 count，不触发任务 API
- [ ] `data:view` / `data:manage` 已写入用户权限与角色矩阵
- [ ] 禁用监控项不出现在夜间统计与 batch recount 中
- [ ] `related_task_ids` 中的 `task_id` 在任务表中存在（允许为空数组）；前端用任务列表解析名称，缺失时回退显示 `task_id`

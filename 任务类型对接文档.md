# 任务管理任务类型对接文档（P0）

本文档面向 **前端任务管理页面**，说明后端 Manager 新增「任务类型」字段后的接口约定。  
本次改动用于支持任务在创建、编辑、列表展示和筛选时按业务类型管理。

| 项目 | 路径 |
| --- | --- |
| 前端页面 | `/tasks`（任务管理页面） |
| 任务列表接口 | `GET /api/tasks` |
| 任务详情接口 | `GET /api/tasks/:taskId` |
| 创建任务接口 | `POST /api/tasks` |
| 编辑任务接口 | `PUT /api/tasks/:taskId` |
| 后端数据表 | `tasks.task_category` |

---

## 0. 现状说明（重要）

| 项 | 现状 |
| --- | --- |
| 任务运行环境 | 已有 `node_type`，表示 Python / Shell / C++ 等运行环境 |
| 业务任务类型 | 之前没有独立字段，前端无法按「运维 / 数据下载 / 因子计算 / 模型训练」分类展示 |
| 创建 / 编辑表单 | 之前不支持选择业务类型 |
| 任务筛选 | 之前不能按业务类型筛选 |

注意：`node_type` 和本次新增的 `task_category` 含义不同。

| 字段 | 含义 | 示例 |
| --- | --- | --- |
| `node_type` | 任务运行环境 | `python` |
| `task_category` | 任务业务类型，前端展示中文 | `数据下载任务` |
| `task_category_code` | 任务业务类型稳定枚举，推荐保存/筛选使用 | `data_download` |

---

## 1. 目标行为

任务管理页面新增「任务类型」能力：

```text
打开任务管理页面
  → GET /api/tasks
  → 每行展示任务类型中文

选择任务类型筛选
  → GET /api/tasks?task_category=xxx
  → 只展示对应类型任务

新建 / 编辑任务
  → 表单中选择任务类型
  → POST /api/tasks 或 PUT /api/tasks/:taskId 提交
```

- 任务类型下拉框展示中文。
- 前端请求推荐传英文枚举 code，后端也兼容中文。
- 老任务未配置类型时，后端默认归为「运维任务」。

## 2. 类型枚举

| 前端展示 | 推荐提交值 | 后端存储值 |
| --- | --- | --- |
| 运维任务 | `ops` | `ops` |
| 数据下载任务 | `data_download` | `data_download` |
| 因子计算任务 | `factor_compute` | `factor_compute` |
| 模型训练任务 | `model_training` | `model_training` |

前端下拉建议：

```ts
const taskCategoryOptions = [
  { label: '运维任务', value: 'ops' },
  { label: '数据下载任务', value: 'data_download' },
  { label: '因子计算任务', value: 'factor_compute' },
  { label: '模型训练任务', value: 'model_training' }
]
```

## 3. 接口变更：任务列表 `GET /api/tasks`

### 3.1 Query 参数

新增按任务类型筛选参数。

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `task_category` | string | 否 | 任务类型筛选，推荐传英文枚举 |
| `taskCategory` | string | 否 | camelCase 兼容字段 |

筛选值支持英文枚举和中文展示值。

**示例**

```http
GET /api/tasks?task_category=data_download
```

```http
GET /api/tasks?taskCategory=factor_compute
```

```http
GET /api/tasks?task_category=因子计算任务
```

### 3.2 响应字段

任务列表每个任务项新增字段：

```json
{
  "task_id": "task_market_stock_daily",
  "name": "Daily Market Data - Stock (All Frequencies)",
  "node_type": "python",
  "task_category": "数据下载任务",
  "taskCategory": "数据下载任务",
  "task_category_code": "data_download",
  "taskCategoryCode": "data_download",
  "task_category_label": "数据下载任务",
  "taskCategoryLabel": "数据下载任务"
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `task_category` | string | 中文展示值，前端表格可直接展示 |
| `taskCategory` | string | camelCase 中文展示值 |
| `task_category_code` | string | 英文枚举，推荐用于表单保存和筛选 |
| `taskCategoryCode` | string | camelCase 英文枚举 |
| `task_category_label` | string | 中文展示值，兼容 label 命名 |
| `taskCategoryLabel` | string | camelCase 中文展示值 |

> 推荐：表格展示用 `task_category`；筛选和提交用 `task_category_code`。

## 4. 接口变更：任务详情 `GET /api/tasks/:taskId`

详情接口返回同样字段，供编辑表单回显。

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task_cj_quant_factors_daily",
    "name": "CJ Quant Factors Daily",
    "node_type": "python",
    "task_category": "因子计算任务",
    "taskCategory": "因子计算任务",
    "task_category_code": "factor_compute",
    "taskCategoryCode": "factor_compute"
  }
}
```

编辑表单回显建议使用：

```ts
form.task_category = data.task_category_code
```

## 5. 接口变更：创建任务 `POST /api/tasks`

### 5.1 请求字段

新增字段：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `task_category` | string | 否 | 任务业务类型，推荐传英文枚举 |
| `taskCategory` | string | 否 | camelCase 兼容字段 |

未传时默认：`ops`（运维任务）。

### 5.2 请求示例

推荐传英文枚举：

```json
{
  "task_id": "task_example_download",
  "name": "Example Data Download",
  "node_type": "python",
  "task_category": "data_download",
  "script_path": "src/scripts/example_download.py",
  "cron_expr": "",
  "timeout_sec": 3600,
  "enabled": 1,
  "target_node_id": "node_001",
  "default_params": {}
}
```

也兼容传中文：

```json
{
  "task_id": "task_example_factor",
  "name": "Example Factor Compute",
  "node_type": "python",
  "taskCategory": "因子计算任务",
  "script_path": "src/scripts/example_factor.py"
}
```

### 5.3 响应

创建成功响应结构不变：

```json
{
  "code": 0,
  "message": "Task created successfully"
}
```

创建成功后前端可刷新列表获取任务类型展示字段。

---

## 6. 接口变更：编辑任务 `PUT /api/tasks/:taskId`

### 6.1 请求字段

编辑接口支持更新任务类型：

```json
{
  "task_category": "model_training"
}
```

或：

```json
{
  "taskCategory": "模型训练任务"
}
```

### 6.2 响应字段

编辑成功后返回任务详情，包含中文展示值和英文 code：

```json
{
  "code": 0,
  "message": "Task updated successfully",
  "data": {
    "task_id": "task_model_train_daily",
    "name": "Model Train Daily",
    "task_category": "模型训练任务",
    "taskCategory": "模型训练任务",
    "task_category_code": "model_training",
    "taskCategoryCode": "model_training"
  }
}
```

## 7. 前端改造清单

| 项 | 说明 |
| --- | --- |
| 任务列表表格 | 新增「任务类型」列，展示 `task_category` 或 `taskCategory` |
| 筛选区 | 新增「任务类型」下拉框，选项见第 2 节 |
| 列表请求 | 选择筛选项后请求 `GET /api/tasks?task_category=<code>` |
| 新建任务弹窗 | 新增任务类型下拉，默认选「运维任务」或按业务决定默认值 |
| 编辑任务弹窗 | 用详情/列表返回的 `task_category_code` 回显下拉值 |
| 保存提交 | 推荐提交英文 code：`task_category: form.task_category` |
| 兼容处理 | 若后端旧版本没有字段，可前端临时默认展示 `运维任务` |

筛选交互建议：

```text
选择任务类型
  → 更新筛选条件
  → 重新请求 GET /api/tasks?task_category=<code>

选择「全部」
  → 不传 task_category 参数
```

---

## 8. 与旧字段的差异

| 字段 | 旧含义 / 新含义 | 是否新增 |
| --- | --- | --- |
| `node_type` | 运行环境，如 `python` | 已有，不变 |
| `task_type` | 调度方式推导值，如 `dag` / `cron` / `manual` | 已有，不变 |
| `task_category` | 业务任务类型中文展示值 | 新增 |
| `task_category_code` | 业务任务类型英文枚举 | 新增 |

不要用 `node_type` 表示「数据下载任务 / 因子计算任务」；这两个概念需要分开展示和筛选。

## 9. 后端落库与迁移说明

后端新增字段：

```sql
ALTER TABLE tasks
ADD COLUMN task_category VARCHAR(32) NOT NULL DEFAULT 'ops'
COMMENT '任务业务类型: ops/data_download/factor_compute/model_training'
AFTER node_type;
```

迁移文件：

```text
quant_ops_system/db/migrations/008_task_category.sql
```

Manager 启动时也会自动检查并补齐 `tasks.task_category` 字段。

现有任务初始分类规则：

| 类型 | 初始归类 |
| --- | --- |
| 行情、基础财务、股票信息、AH、ST、指数成分、资产配置 | 数据下载任务 |
| 派生指标、Wind 一致预期、CJ/FZ/Vendor 因子 | 因子计算任务 |
| 测试任务、数据监控统计任务 | 运维任务 |
| 模型训练任务 | 当前暂无预置任务，后续创建时选择 |
